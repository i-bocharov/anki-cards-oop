# 🧠 Anki CLI — тренировка слов

Консольное приложение для изучения слов с поддержкой разных режимов тренировки, загрузки данных из файлов и сети.

Проект реализует упрощённую версию Anki с акцентом на чистую архитектуру, расширяемость и строгую типизацию.

---

## 🚀 Возможности

* 📚 Хранение слов (слово → перевод)
* 🎮 Игровой режим перевода слов
* 🧪 Режимы тренировки:

  * До первой ошибки
  * На время
* 📂 Поддержка разных источников данных:

  * `.txt` (CSV)
  * `.tsv`
  * `.json`
  * HTTP(S) JSON API
* 🧠 Нормализация данных (lowercase + trim)
* 🔌 Расширяемая система загрузчиков (через реестр)
* 🛡️ Валидация входных данных
* 🧪 Покрытие тестами (unit + integration)


Или с указанием источника:


## 🕹️ Использование

После запуска откроется меню:

```
1. Начать игру
2. Добавить слова
3. Тренировка до первой ошибки
4. Тренировка на время
5. Показать все слова
6. Найти перевод
7. Выход
```

### ⛔ Выход из режимов

Во время ввода можно ввести:

```
СТОП
```

## 🧱 Архитектура

### Основные компоненты

#### `Anki`

* Управляет словарём
* Валидация и нормализация
* Бизнес-логика

#### `TrainingSession`

* Базовый класс тренировки
* Управляет состоянием (время, счёт)

Реализации:

* `ZeroMistakesTraining`
* `TimeLimitedTraining`

#### `LoaderProtocol`

Контракт загрузчиков:

```python
load_words() -> dict[str, str]
save_words(words: dict[str, str]) -> None
```

#### `LoaderRegistry`

* Реестр загрузчиков
* Автоматический выбор loader по source

#### Загрузчики

* `TextFileLoader`
* `TSVFileLoader`
* `JsonFileLoader`
* `JsonNetworkLoader`

#### `TextUI`

* CLI интерфейс
* Меню и взаимодействие с пользователем

#### `GameContext`

* Context Manager
* Загружает слова при входе
* Сохраняет при выходе (даже при ошибках)

## ⚡ Сложность алгоритмов

Обозначения:

* `n` — количество слов
* `k` — длина строки

### Класс `Anki`

| Операция               | Сложность | Пояснение                             |
| ---------------------- | --------- | ------------------------------------- |
| `add_word`             | O(k)      | нормализация строки                   |
| `normalize_word`       | O(k)      | `strip + lower`                       |
| `get_random_word`      | O(n)      | преобразование `dict.keys()` в список |
| `get_random_word_pair` | O(n)      | аналогично                            |
| `check_translation`    | O(k)      | нормализация + O(1) lookup            |
| `get_translation`      | O(k)      | нормализация + O(1) lookup            |
| `__contains__`         | O(k)      | нормализация + O(1)                   |
| `__len__`              | O(1)      | длина dict                            |
| `__iter__`             | O(n)      | итерация по dict                      |

💡 Замечание:
`random.choice(list(...))` создаёт список → O(n).
Можно оптимизировать через `tuple` кэш или хранение списка ключей.

---

### TrainingSession

| Операция            | Сложность | Пояснение          |
| ------------------- | --------- | ------------------ |
| `get_random_word`   | O(n)      | делегирует Anki    |
| `check_translation` | O(k)      | делегирует Anki    |
| `get_stat`          | O(1)      | простая арифметика |

---

### Loader'ы

#### Загрузка

| Loader    | Сложность              |
| --------- | ---------------------- |
| TXT / TSV | O(n)                   |
| JSON      | O(n)                   |
| Network   | O(n) + network latency |

#### Сохранение

| Loader    | Сложность |
| --------- | --------- |
| TXT / TSV | O(n)      |
| JSON      | O(n)      |

---

### Память

| Компонент           | Сложность                |
| ------------------- | ------------------------ |
| Хранение слов       | O(n)                     |
| Временные структуры | O(n) (при random.choice) |
